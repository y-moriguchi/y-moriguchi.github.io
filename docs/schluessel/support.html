機能

    • SRFI-4 (一様なベクタ)

    • SRFI-7
    • SRFI-8 (receive)
    • SRFI-9 (レコード型)
    • SRFI-10 (Sharp-comma external forms)
    • SRFI-13 (文字列ライブラリ)
    • SRFI-14 (文字セット)
    • SRFI-16 (Syntax for procedures of variable arity)
    • SRFI-17 (一般化されたset!)
    • SRFI-18 (マルチスレッド)
    • SRFI-19 (日付オブジェクト)
    • SRFI-23 (エラーレポート)
    • SRFI-25 (多次元配列)
    • SRFI-28 (文字列のフォーマット)
    • SRFI-30 (ネストされたコメント)
    • SRFI-34 (例外処理)
    • SRFI-37
    • SRFI-38 (共有構造の表現)
    • SRFI-39 (パラメータ)
    • SRFI-41 (ストリーム)
    • SRFI-44 (コレクション)
    • SRFI-47 (配列)
    • SRFI-58 (配列の記法)
    • SRFI-59
    • SRFI-60 (ビットとしての整数)
    • SRFI-62 (S式コメント)
    • SRFI-63 (一様および一様でない配列)
    • SRFI-66 (オクテットベクター)
    • SRFI-67 (比較手続き)
    • SRFI-69 (基本的なハッシュ表)
    • SRFI-94 (型の制限された数値関数)
    • SRFI-98 (環境変数へのアクセス)
  • マルチバイト文字列が使用可能です
  • 10進浮動小数点(Decimal32, Decimal64)が使用可能です
  • ClojureライクなJavaライブラリの参照が可能です
  • Javaライブラリをインポートして使用することが可能です
  • Javaクラスをレコードのように使用することが可能です
  • Java側からSchemeのコードの実行・Schemeの変数の取得・Schemeの手続きの実行が可能です
  • #構文をユーザで定義することが可能です
  • 形式言語によるパーサを構築することが可能です。LALR(1)の言語クラスが使用可能です
  • 正規表現による字句解析器(トークナイザ)が構築することが可能です
  • Schluesselプラグイン機構
  • サウンド機能
  • 総称関数と簡易的なObjectSystemをサポートしています
  • syntax-case形式のマクロの一部をサポートしています。サポートされている機能は以下の通りです
    • syntax-case
    • syntax(#'), quasisyntax(#`), unsyntax(#,), unsyntax-splicing(#,@)
    • make-variable-transformer
    • identifier?
    • syntax->datum, datum->syntax
    • generate-tempolaries
  • 図形言語・グラフィックス機能をサポートしています。現在サポートしている機能は以下の通りです。
    • 画像の入力・描画
    • 線・矩形・楕円・文字の描画
    • 画像の出力(JPEG, GIF, PNGをサポートしています)
  • 特殊関数の計算が可能です。サポートしている特殊関数は以下の通りです
    • Bessel関数(複素数領域・次数は任意の実数)
    • ガンマ関数(複素数領域)
  • JDBCを使用したRDBとの連携が可能です
  • Servletを応用したサーバサイドアプリケーションを記述可能です
  • XMLを解析可能です
  • AppletからSchemeを起動可能です
  • Swingとの連携が可能です
  • ファイルの操作が可能です
  • 正規表現が使用可能です
  • 線形代数ライブラリ(行列の基本演算・LU分解・連立１次方程式の解法等)が使用可能です
  • 正確な複素数の演算が可能です
  • 四元数/八元数が使用可能です
  • 1変数多項式の計算が可能です
  • 実数の区間ライブラリが使用可能です
  • ネットワーク手続きが使用可能です
  • HTTP, FTP, SMTP, SNMP, LDAPが使用可能です
  • JNDIが使用可能です
  • zipファイルの読み込みとgzipファイルの読み書きが使用可能です

  • プロセスが使用可能です

制限事項
  • syntax-caseの制限
    • 以下のような引数に展開してパターンにするようなマクロを記述することができません
      • (define-syntax with-syntax
          (lambda (x)
            (syntax-case x ()
              ((_ ((p e0) ...) e1 e2 ...)
               (syntax (syntax-case (list e0 ...) ()
                         ((p ...) (let () e1 e2 ...))))))))
