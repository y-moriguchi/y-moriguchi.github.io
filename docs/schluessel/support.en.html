Features
  • Supports R5RS features
  • Supports R5RS macro and traditional macro
  • Supports arbitrary precision integers, flonum, and complex numbers
  • Supports SRFIs (Scheme Request for Implementation) as follows
    • SRFI-0 (Feature-based conditional expansion construct)
    • SRFI-1 (List libraries)
    • SRFI-2 (AND-LET*)
    • SRFI-4 (Homogenious vectors)
    • SRFI-6 (String ports)
    • SRFI-8 (receive)
    • SRFI-9 (records)
    • SRFI-10 (Sharp-comma external forms)
    • SRFI-13 (String libraries)
    • SRFI-14 (Character sets)
    • SRFI-16 (Syntax for procedures of variable arity)
    • SRFI-17 (Generazied set!)
    • SRFI-18 (Multithreading)
    • SRFI-19 (Time Data Types and Procedures)
    • SRFI-22 (Running Scheme Scripts on Unix)
    • SRFI-23 (Error reports)
    • SRFI-25 (Multi-dimensional Array Primitives)
    • SRFI-28 (Basic Format Strings)
    • SRFI-30 (Nested Multi-line Comments)
    • SRFI-34 (Exception Handling for Programs)
    • SRFI-38 (External Representation for Data With Shared Structure)
    • SRFI-39 (Parameter objects)
    • SRFI-41 (Streams)
    • SRFI-44 (Collections)
    • SRFI-47 (Array)
    • SRFI-58 (Array Notation)
    • SRFI-60 (Integers as Bits)
    • SRFI-62 (S-expression comments)
    • SRFI-63 (Homogeneous and Heterogeneous Arrays)
    • SRFI-66 (Octet Vectors)
    • SRFI-67 (Compare Procedures)
    • SRFI-69 (Basic hash tables)
    • SRFI-94 (Type-Restricted Numerical Functions)
    • SRFI-98 (An interface to access environment variables)
  • Supports multi-byte characters and strings
  • Supports Decimal32 and Decimal64
  • Supports Java library reflection, which has Clojure-like syntax.
  • Java classes can be used as records
  • Scheme codes manipulation by Java
  • User-defined #-syntax
  • Building parsers with formal language; LALR(1) language class supported
  • Building lexical analyser (tokenizer) with regular expression
  • Schluessel plugin
  • Sound libraries
  • Supports generic functions and tiny ObjectSystem
  • Supports syntax-case macro partially. Supported features are listed as follows
    • syntax-case
    • syntax(#'), quasisyntax(#`), unsyntax(#,), unsyntax-splicing(#,@)
    • make-variable-transformer
    • identifier?
    • syntax->datum, datum->syntax
    • generate-tempolaries
  • Supports graphics language. Supported features are listed as follows
    • inputs from image file
    • draws line, rectangle, ellipsis, characters
    • outputs to image file (JPEG, GIF and PNG are supported)
  • Computing special functions. Supported functions are listed as follows
    • Bessel functions(complex domain)
    • Gamma functions(complex domain)
  • Supports accessing RDBs used by JDBC
  • Web applications using Java Servlet technology
  • Supports XML Parsing
  • Applet by Scheme
  • Supports Swing applications
  • Supports file manipulation
  • Supports regular expressions
  • Supports linear algebra library(elementary arithmetics of matrices, LU decomposition, solving linear equations)
  • Supports exact complex numbers
  • Supports quaternions and octionions
  • Supports 1-variable polynomials
  • Supports ranges of real number
  • Supports networking
  • HTTP, FTP, SMTP, SNMP, LDAP are avaliable
  • JNDI is avaliable
  • Supports reading zip files and reading and writing gzip files
  • Supports desktop's tray icons
  • R6RS I/O is available
  • Statistics procedures are available
  • Exact pi and Nepier's number are available
  • An iterator framework is available
  • Processes are available
  • Supports turtle graphics
Restrictions
  • restriction of syntax-case
    • with-syntax is not supported
      • (define-syntax with-syntax
          (lambda (x)
            (syntax-case x ()
              ((_ ((p e0) ...) e1 e2 ...)
               (syntax (syntax-case (list e0 ...) ()
                         ((p ...) (let () e1 e2 ...))))))))
