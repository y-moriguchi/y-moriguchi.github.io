5.1 ファイル操作

ファイルを操作する手続きです。

Function: chdir [dir]

戻り値: 引数がないときは現在の作業ディレクトリ、引数があるときは未定義
現在の作業ディレクトリを変更します。

Function: directory? path

戻り値: ディレクトリならば#t
与えられたパスが存在するディレクトリであるかを調べます。

Function: file? path

戻り値: ファイルならば#t
与えられたパスが存在するファイルであるかを調べます。

Function: exist? path

戻り値: 存在すれば#t
与えられたパスが存在するかを調べます。

Function: directory-list dir [wildcard-filter-expr]

戻り値: ディレクトリに存在するファイルのリスト
ディレクトリに存在するファイルを取得します。この手続きはディレクトリを再帰的に評価しません。
wildcard-filter-exprには以下の式が指定できます。

(and [expr …])与えられた条件をすべてみたすとき選択されます
(or [expr …])与えられた条件のいずれかをみたすとき選択されます
(not expr)与えられた条件を満たさないとき選択されます
exist存在するパスを選択します
directory存在するディレクトリを選択します
file存在するファイルを選択します
文字列文字列で与えられたワイルドカードに一致するパスを選択します。 ワイルドカードのメタ文字は次の通りです。
*任意の0文字以上
.任意の1文字
[…]文字集合
Function: directory-fold path proc seed [wild-card]

戻り値: ファイルならば#t
与えられたパスのサブディレクトリにある(ワイルドカードでフィルタされた)ファイル名すべてに対してfoldします。 procは2つの引数を持ち、第1引数はファイル名、第2引数は今までの結果が与えられます。

Function: make-path-name path file

戻り値: 新しいパス
パス名とファイル名を組み合わせて新しいパスを生成します。この手続きを使用すればOSのパス区切りの依存性を回避することができます。

Function: expand-path path

戻り値: 文字列
パスを絶対パスに変換します。

Function: absolute-path? path

戻り値: パスが絶対パスのとき#t
パスが絶対パスかを調べます。

Function: relative-path? path

戻り値: パスが相対パスのとき#t
パスが相対パスかを調べます。

Function: file-mtime=? time1 time2

戻り値: 下記参照
ファイルの更新日時が等しいかを調べます。 引数にはファイルパス・SRFI-19形式の時間オブジェクト・UNIXエポックからの秒数指定できます。

Function: file-mtime>=? time1 time2

戻り値: 下記参照
ファイルの更新日時が同じまたは未来であるかを調べます。

Function: file-mtime>? time1 time2

戻り値: 下記参照
ファイルの更新日時が未来であるかを調べます。

Function: file-mtime<=? time1 time2

戻り値: 下記参照
ファイルの更新日時が同じまたは過去であるかを調べます。

Function: file-mtime<? time1 time2

戻り値: 下記参照
ファイルの更新日時が過去であるかを調べます。

Function: mkdir path

戻り値: ディレクトリが作成されれば#t
ディレクトリを作成します。

Function: copy-file src-file dest-file

戻り値: 未定義
ファイルをコピーします。

Function: move-file src-file dest-file

戻り値: 移動が正常終了すれば#t ファイルを移動します。

Function: remove-file file …

戻り値: 未定義
ファイルを削除します。

Function: rmdir path

戻り値: ディレクトリを削除できれば#t
ディレクトリを削除します。

Function: touch-file file …

戻り値: 未定義
ファイルをtouchします。

Function: chmod mode file …

ファイルの権限を変更します。modeはUNIXのchmod(1)に似た次の2形式が可能です。
1. 文字による指定
abcの3文字によって指定します。文字は以下の指定が可能です。
1文字目:

u実行しているユーザ
aすべてのユーザ

2文字目:

+3文字目の権限を追加する
-3文字目の権限を削除する

3文字目:

r読み込み権限
w書き込み権限
x実行権限

2. 数字による指定
数字2桁により指定します。2桁目は実行しているユーザ、1桁目はすべてのユーザを変更します。 (UNIXのchmod(1)では3桁ですが、この手続きでは2桁です。)
数字のビットは以下の意味を持ちます。

1実行権限
2書き込み権限
4読み込み権限
Function: partition-free-space path
Function: partition-total-space path
Function: partition-usable-space path

それぞれパーティションの空き領域、全領域、使用可能な領域を取得します。
